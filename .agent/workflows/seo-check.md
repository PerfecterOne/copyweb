# 页面 SEO 检查标准流程 (SOP)

本工作流用于指导开发者在发布页面前，系统性地检查页面是否满足核心 SEO 最佳实践。

## 检查清单

### 1. Title (页面标题) 检查
- [ ] **独特性**：每个页面是否有唯一的标题？
- [ ] **长度控制**：标题长度是否在 50-60 个字符以内（避免被搜索结果截断）？
- [ ] **关键词顺序**：重要的关键词是否放在标题首位？且读起来自然？
- [ ] **品牌名称**：标题中是否包含了品牌名称？

### 2. Description (页面描述) 检查
- [ ] **独特性**：该页面的描述是否清晰反映了该页面的独特价值？
- [ ] **长度控制**：长度是否在 150-160 个字符以内（包括空格）？
- [ ] **内容质量**：是否包含核心关键词？是否避免了关键词堆砌？是否是一两个自然的句子？
- [ ] **号召行动 (CTA)**：是否包含了如“学习”、“购买”、“免费使用”等引导性词汇？

### 3. Heading (标题标签 H1-H6) 检查
- [ ] **H1 唯一性**：页面是否**有且仅有一个** H1 标签？
- [ ] **相关性**：H1 是否准确概括了页面主题？H2-H6 是否逻辑清晰地划分了内容段落？
- [ ] **可读性**：标题是否反映了文本情感，避免了单纯的数字序号？

### 4. 图片 Alt 属性检查
- [ ] **完整性**：所有关键图片（产品、图表、头图）是否有 `alt` 属性？
- [ ] **描述性**：`alt` 内容是否简练地描述了图片内容，而不仅仅是堆砌关键词？

### 5. 链接的 Nofollow 属性检查
- [ ] **外部链接处理**：对于不受信任的内容、赞助链接或用户生成内容中的外链，是否添加了 `rel="noopener noreferrer nofollow"`？
- [ ] **内部链接**：确保内部链接**没有**错误地使用 nofollow。

### 6. Robots 元标签检查
- [ ] **索引逻辑**：对于内容稀少或未完成的页面，是否配置了 `<meta name="robots" content="noindex">`？
- [ ] **一致性**：检查是否错误地限制了重要页面的索引。

### 7. Canonical (规范网址) 标签检查
- [ ] **准确性**：`<link rel="canonical" href="...">` 指向的地址是否为该页面的标准唯一 URL？
- [ ] **参数过滤**：确保带跟踪参数（如 `?ref=...`）的访问也会通过规范标签将权重聚拢。

### 8. Schema Markup (模式标记) 检查
- [ ] **JSON-LD 格式**：页面头部是否包含符合谷歌要求的 JSON-LD 结构化数据（如 Product, Article, FAQ 等）？
- [ ] **数据完整性**：标记的数据内容是否准确（名称、图片、描述等）？

### 9. Social Media (社媒元标签) 检查
- [ ] **OG 标签**：是否配置了 `og:title`, `og:url`, `og:description`, `og:image` 等？
- [ ] **Twitter Cards**：是否额外针对 Twitter 进行了适配？

### 10. Viewport (视口元标签) 检查
- [ ] **适配配置**：是否存在 `<meta name="viewport" content="width=device-width, initial-scale=1"/>` 以确保移动端适配？

---

## 检查建议
1. 使用 **Detailed SEO Extension** 或类似插件进行可视化检查。
2. 重点对照竞争对手的 Title 和 Description 写作技巧。
3. 确保所有改动在上线前均已通过代码审查。
